Section A — Analyse automatique (impeccable)

Automatique
Warning
Side-tab accent border
La bordure gauche colorée de .copy-quote est l'un des patterns les plus reconnaissables des UI générées par IA. Elle signale "quelque chose d'important" mais par répétition devient un tic visuel.
/* Ligne détectée dans index.html */ .copy-quote { border-left: 3px solid var(--vert); /* ← side-tab */ background: var(--vert-pale); border-radius: 0 8px 8px 0; padding: 10px 12px; }
Avant — cliché IA
« Grâce à mon intervention, l'équipe a économisé 2 jours de blocage. »
Après — bordure complète subtile
« Grâce à mon intervention, l'équipe a économisé 2 jours de blocage. »
/* Fix proposé : supprimer la bordure gauche seule, utiliser une bordure complète discrète */ .copy-quote { border: 1px solid rgba(0,105,70,0.25); background: rgba(0,105,70,0.06); border-radius: 8px; /* rayon uniforme */ padding: 10px 12px; /* supprimer border-left: 3px + border-radius: 0 8px 8px 0 */ }
Warning
Wide letter-spacing sur texte courant
Un letter-spacing: 0.10em est appliqué sur .section-tag — classe utilisée comme label de section lisible, pas comme un sigle monogramme. Au-delà de 0.05em, l'espacement fragmente les mots et ralentit la lecture.
/* Ligne détectée dans index.html */ .section-tag { font-size: 0.75rem; font-weight: 700; letter-spacing: 0.1em; /* ← wide-tracking sur body text */ padding: 4px 10px; border-radius: 6px; }
Avant — tracking 0.10em
Ce qui s'est passé
Après — tracking 0.05em
Ce qui s'est passé
/* Fix : réduire l'espacement de moitié */ .section-tag { letter-spacing: 0.05em; }

Section B — Critique UI/UX manuelle

Manuel
Medium
Texte résiduel sous 12 px
.copy-label (0.65rem ≈ 10.4 px) et .step-badge (0.7rem ≈ 11.2 px) sont en dessous du seuil WCAG de lisibilité minimale (12 px). Ces éléments ont échappé à la passe typo précédente.
/* index.html — CSS actuel */ .copy-label { font-size: 0.65rem; /* ≈ 10.4 px — sous le seuil */ font-weight: 700; letter-spacing: 0.08em; text-transform: uppercase; color: var(--vert); } .step-badge { font-size: 0.7rem; /* ≈ 11.2 px — sous le seuil */ font-weight: 700; /* ... */ }
Avant — illisible
Partage ton impact
1 Ce qui s'est passé
Après — 0.75rem minimum
Partage ton impact
1 Ce qui s'est passé
/* Fix : aligner sur le niveau "méta" minimal */ .copy-label { font-size: 0.75rem; } .step-badge { font-size: 0.75rem; }
Medium
Classes CSS nommées --rouge qui rendent en vert
Suite à la migration sémantique (finding 11), .field-value--rouge et .pill--rouge utilisent désormais var(--vert). Le rendu est correct, mais le nom trahi l'intention et rend le CSS impossible à maintenir pour quelqu'un d'autre.
/* index.html — dette de nommage */ .field-value--rouge { color: var(--vert); } /* ← "rouge" = vert */ .pill--rouge[aria-pressed="true"] { background: var(--vert); /* ← idem */ border-color: var(--vert); }
Avant — nom mensonger
3 heures ← classe .pill--rouge
rendu vert
Après — nom cohérent
3 heures ← classe .pill--vert
rendu vert ✓
/* Fix : renommer les sélecteurs et les références HTML */ .field-value--vert { color: var(--vert); } .pill--vert[aria-pressed="true"] { background: var(--vert); border-color: var(--vert); } /* + mettre à jour les pills.classList.add() dans le JS (ligne ~460) */
Low
Variables CSS orphelines — --rouge-* inutilisées
Trois variables définies dans :root ne sont plus utilisées nulle part dans le CSS ou le JS après la migration couleur : --rouge, --rouge-pale, --rouge-border. Elles allourdissent la feuille de style et brouillent la palette officielle.
/* :root dans index.html — variables mortes */ :root { /* ... */ --rouge: #C0392B; /* ← 0 usage */ --rouge-pale: rgba(192,57,43,0.05); /* ← 0 usage */ --rouge-border: rgba(192,57,43,0.12); /* ← 0 usage */ }
/* Fix : supprimer les 3 lignes de :root Vérifier d'abord : grep --rouge index.html Résultat attendu : 0 occurrence hors :root */ :root { /* --rouge supprimé */ /* --rouge-pale supprimé */ /* --rouge-border supprimé */ }
Medium
Fond --vert-pale trop transparent — badge de section invisible
--vert-pale: rgba(0,105,70,0.06) produit un fond à 6 % d'opacité sur blanc : le contraste du fond lui-même est imperceptible. Les section-tags et la tile d'estimation utilisent cette valeur — leur affordance visuelle "zone verte" est perdue.
/* :root */ --vert-pale: rgba(0,105,70,0.06); /* 6 % — quasi-invisible */ /* Utilisateurs : */ .section-tag--factuel { background: var(--vert-pale); } .section-tag--estimation{ background: var(--vert-pale); } .tile--estimation { background: var(--vert-pale); }
Avant — 6% opacité
Ce qui s'est passé
Fond à 6% sur blanc — indiscernable
Après — 12% + bordure
Ce qui s'est passé
Fond à 12% + bordure — zone identifiable
/* Fix : doubler l'opacité + ajouter une bordure sur les tags */ --vert-pale: rgba(0,105,70,0.10); /* 10–12% lisible */ .section-tag--factuel, .section-tag--estimation { border: 1px solid var(--vert-border); }
High
Absence de styles :focus-visible — navigation clavier impossible
Aucune règle :focus-visible n'est définie dans la feuille de style. Les utilisateurs clavier (Tab, Shift+Tab) ne voient aucun indicateur de focus sur les pills, boutons devise, sliders ou boutons copie. Violation directe de WCAG 2.4.7 (Focus Visible, niveau AA).
/* index.html — aucun focus défini */ /* grep ":focus" index.html → 0 résultat */ /* Éléments interactifs sans focus visible : */ .pill /* sélection de durée */ .devise-btn /* EUR / CAD */ .copy-btn /* copier la phrase */ input[type="range"] /* sliders */
Avant — focus invisible
Cliquer → aucun anneau de focus. Navigateur masque l'outline par défaut.
Après — focus jaune visible
Tab → anneau jaune bien visible sur fond sombre.
/* Fix : ajouter en bas du bloc CSS */ :focus-visible { outline: 3px solid var(--jaune); outline-offset: 3px; border-radius: 4px; } /* Optionnel : neutraliser sur les sliders (custom thumb déjà visible) */ input[type="range"]:focus-visible { outline: none; } input[type="range"]:focus-visible::-webkit-slider-thumb { box-shadow: 0 0 0 3px var(--jaune); }
